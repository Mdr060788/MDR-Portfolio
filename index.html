Violencia Familiar — Línea 137

Power BI Figma Datos Abiertos Sector Público
🎯 El desafío

Los datos sobre llamados a la Línea 137 en el AMBA estaban disponibles públicamente pero en formato crudo, sin ninguna visualización que permitiera entender patrones, zonas críticas o evolución temporal. El objetivo fue transformar ese dataset en algo que cualquier persona pudiera leer e interpretar.

⚙️ El proceso

Los datos se obtuvieron directamente desde el portal de datos abiertos del gobierno argentino. Antes de construir el reporte, diseñé el layout completo en Figma — una práctica que mejora enormemente el resultado final porque separás las decisiones de diseño de las técnicas. La transformación y modelado se realizaron en Power Query dentro de Power BI.

✦ El resultado

Un reporte interactivo que permite explorar los llamados por zona geográfica, franja horaria y evolución mensual. Datos públicos convertidos en insight social concreto.

Licitaciones Viales Nacionales

Python Selenium Power BI Geolocalización Sector Público
🎯 El desafío

La información sobre licitaciones viales nacionales estaba dispersa en el sitio oficial del gobierno, sin una forma simple de consultarla, filtrarla ni analizarla en conjunto. Seguir el estado de cada obra requería revisión manual constante.

⚙️ El proceso

Desarrollamos un scraper con Python y Selenium para extraer automáticamente todas las licitaciones publicadas, incluyendo estado, presupuesto y plazos. Los datos se limpiaron y normalizaron para garantizar precisión. Luego usamos un plugin de geolocalización para inferir las coordenadas de cada obra a partir de las descripciones textuales, permitiendo mapearlas geográficamente.

✦ El resultado

Un reporte en Power BI con mapa interactivo, filtros por estado y provincia, evolución de presupuestos y seguimiento de plazos. Monitoreo automático de obra pública sin intervención manual.

Predicción de Abandono de Clientes

Python PyCaret Streamlit Power BI Looker Studio Machine Learning
🎯 El desafío

Retener un cliente existente cuesta hasta 5 veces menos que conseguir uno nuevo. El problema es identificarlo antes de que se vaya. Este proyecto busca exactamente eso: predecir qué clientes tienen alta probabilidad de abandono para actuar de forma preventiva.

⚙️ El proceso

El modelo fue entrenado con el dataset público Churn Bank usando PyCaret en Python, que permitió comparar múltiples algoritmos y seleccionar el de mejor performance automáticamente. La app se desplegó en Streamlit e integra reportes tanto en Power BI como en Looker Studio para el análisis exploratorio.

✦ El resultado

Una app funcional con dos modos de uso: predicción individual ingresando los datos de un cliente específico, y predicción por lotes cargando un archivo CSV con múltiples clientes. Lista para ser adaptada a datos reales de cualquier empresa.

BI · Deporte · Modelo propio

Índice de Beneficios para Jugadores

Power BI Geolocalización Modelo Propio Pelota a Paleta Toma de Decisiones
🎯 El desafío

En una organización de torneos de Pelota a Paleta surgió una pregunta aparentemente simple: ¿cómo beneficiar a jugadores que viajan largas distancias? El criterio inicial era claro — más de 200 km promedio recibía bonificación. Pero al analizar los datos apareció un caso que rompió el esquema: dos jugadores con una sola participación de 800 km calificaban para el beneficio máximo. ¿Es justo equiparar eso con quien viaja 150 km pero participa en 10 torneos?

⚙️ El proceso

En lugar de parchear el caso con excepciones, se rediseñó la métrica desde cero. Se construyó un Índice de Esfuerzo multidimensional que combina tres variables: km totales acumulados (35%), percentil 90 de distancias — picos de esfuerzo — (25%), y cantidad de eventos participados — constancia — (40%). Las coordenadas geográficas se obtuvieron y validaron con una extensión de geolocalización para calcular distancias reales entre jugadores y sedes.

📊 Los números

Método original: 23 beneficiados con criterio de umbral fijo. Método nuevo: 27 beneficiados con índice multidimensional. El cambio rescató 26 jugadores sub-beneficiados ignorados por el criterio anterior, y eliminó 21 asignaciones cuestionables basadas solo en distancia. Tasa de acuerdo entre métodos: 86.2%.

✦ El resultado

Un sistema de beneficios por percentil de esfuerzo: ≥97 → 3 pases, ≥95 → 2 pases, ≥92 → 1 pase. No se trata de sofisticación — se trata de tomar decisiones justas, guiadas por datos.

Suite Financiera — Análisis, Forecast & Backtesting

Python LLM Web Scraping APIs Financieras Streamlit Machine Learning
🎯 El desafío

Tomar decisiones de inversión informadas requiere analizar fundamentals, noticias, precios históricos, riesgo y rendimiento comparado — todo al mismo tiempo. Herramientas profesionales para esto son costosas o fragmentadas. El objetivo fue construir una suite completa desde cero, en Python.

⚙️ El proceso

La app se alimenta de APIs financieras y web scraping para obtener precios, fundamentals y noticias en tiempo real. El módulo de análisis fundamental usa LLM para interpretar los datos cualitativos del instrumento. El backtester permite probar diferentes estrategias sobre datos históricos, mientras que el módulo de optimización de portafolio calcula la combinación óptima de activos según perfil de riesgo.

✦ Módulos principales

📈 Backtesting de estrategias de trading
🔮 Predicción de precios con ML
⚖️ Medidores de riesgo y comparación con benchmarks
🧠 Análisis fundamental con LLM
